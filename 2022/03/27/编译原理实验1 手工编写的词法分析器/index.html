编译原理实验1 手工编写的词法分析器

一、实验目的

掌握词法分析器的构造原理，掌握手工设计词法分析器的方法。

二、实验内容

编写一个词法分析器，能够把输入的源程序转换为词法单元序列输出。

三、实验指南

1．采用手工方式设计词法分析器，采用C语言编写。

2．源语言定义见教材附录 A.1，其中的每个终结符即词法分析需要得到的token name。

（1）该语言的关键字：if while do break float true false int char bool float (其中，int、char、bool、float在产生式中为basic)。这些关键字都是小写的保留字。

（2）标识符（id）和数值（num）的正则表达式定义；

（3）专用符号：= + - * / < <= > >= == != ; , ( ) [ ] { }

（4）空白由空格、换行符和制表符组成。空白通常被忽略（它主要用于分开I D、N U M、关键字）。

（5）考虑注释。注释由/和/包含。注释可以放在任何空白出现的位置，且可以超过一行。注释不能嵌套。

​ 3．实现词法分析器的注意要点：

（1） 关键字和标识符名的区别；

（2） 数字的转换处理

（3） “>=”和“>”这类单词的处理；

四、实验步骤

1、处理数字

​ 利用ctype库中的isdigit()函数来检测从输入流中来的字符是否为数字，再进一步对数字整合处理。

2、处理ID

​ 利用ctype库中的isalpha()函数来检测从输入流中来的第一个字符是否为字母，在检测后来的字符是否为数字与字母的组合（因变量名规则是开头为字母的数字字母的组合）。

3、处理关键字

​ 在处理ID的过程中，如果最后得到的字符串和某个关键字匹配，则有理由认为此时的字符串为关键词（因为变量名应当避免与关键字同名）。

4、处理注释
 处理注释主要是分步骤来解决：首先需要看识别的前两个字符是不是`/*`，如果是，则认为后续出现的字符为注释内容，若后续字符连续出现`*` 、`/`，则表明注释内容结束。
5、词法单元的输出

​ 在前四步处理时，一边处理一边输出。处理的结果得到的是一个<int, string>型的pair。只需要对这个pair的第一项进行分类识别，便可对词法单元进行识别输出。
